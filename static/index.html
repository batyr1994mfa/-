Режим размышлений (thinking)
Если включено, модель сначала "рассуждает" (часто на английском, даже если вы просите отвечать по-русски), а потом даёт ответ. Выключите, если нужны короткие прямые ответы без внутренних рассуждений в тексте.
Звуки действий ИИ
Тихий щелчок клавиш пока ИИ печатает, отдельные звуки на начало работы с инструментом, успех, ошибку и завершение ответа — чтобы было понятнее, что процесс идёт, не глядя в экран.
Google-поиск (google_search)
Без ключа инструмент google_search автоматически использует DuckDuckGo. Чтобы ИИ реально искал через Google — вставьте бесплатный API-ключ (console.cloud.google.com → Custom Search API) и ID поисковой системы (programmablesearchengine.google.com).
Почта (send_email)
SMTP-данные для отправки писем от имени ИИ (например, для Gmail: smtp.gmail.com, порт 587, и пароль приложения, а не обычный пароль аккаунта).
Telegram (telegram_send)
Создайте бота через @BotFather, вставьте его токен и chat_id (свой, узнать можно у @userinfobot), чтобы ИИ мог писать вам в Telegram.
Push-уведомления (notify)
Придумайте уникальное имя темы (topic) и подпишитесь на него в приложении ntfy или на сайте ntfy.sh/<тема> — ИИ сможет присылать туда уведомления и напоминания (set_reminder).
Архив решений (save_solution)
Куда ИИ сохраняет успешно решённые технические задачи (по одному JSON-файлу на задачу) и откуда автоматически подмешивает похожие прошлые решения в начало КАЖДОГО сообщения — без того, чтобы модель сама не забыла проверить архив. По умолчанию: ~/AI_memory/solutions.
Приоритет источников знаний
В каком порядке ИИ должен проверять источники, прежде чем отвечать на технический вопрос: сначала 1-е место, и только если там нет ответа — переходит дальше.

🛡 Включить автономность ИИ?

В этом режиме модель получает инструмент run_shell_command — она сможет сама выполнять команды в системном терминале этого компьютера, при необходимости с правами администратора (sudo), используя пароль sudo, сохранённый в настройках. Это выходит далеко за пределы подключённых папок и затрагивает всю систему.
Включайте, только если доверяете модели и понимаете риски: неверная или вредоносная команда может необратимо повредить систему. Каждый вызов по умолчанию требует вашего подтверждения (если не включено автоподтверждение в настройках).
Пароль root (для sudo)
Сайт умеет запоминать пароль автоматически при первом успешном использовании (например, при остановке Ollama), но здесь его можно задать вручную заранее — этим паролем ИИ будет пользоваться для команд с sudo.

🔓 Дать ИИ полный доступ к системе?

Сейчас инструменты чтения/записи файлов (read_file, write_file, list_dir и т.п.) работают только внутри подключённых папок. Этот переключатель добавляет корневую папку / в список подключённых — после этого ИИ сможет читать и создавать/изменять файлы в любом месте файловой системы, а не только в выбранных вами папках.
Это отдельная настройка от «Автономности ИИ» (там речь о выполнении команд в терминале) — здесь именно про файловый доступ. Запись/редактирование по-прежнему требует вашего подтверждения (если не включено автоподтверждение). Отключить можно в любой момент, удалив «/» из списка подключённых папок ниже.
